GitHub → Jenkins → Docker → Argo CD → Kubernetes

Sample Website Deployed on Kubernetes

This demo shows an automated CI/CD workflow. When code is pushed to GitHub, Jenkins scans the code, builds a Docker image, pushes it to a registry, updates the Kubernetes manifest, and Argo CD deploys it to the cluster.

View pipeline Deployment status

Developer commits website changes. A webhook triggers the Jenkins pipeline.

2

SonarQube

Jenkins runs code quality checks before building the application image.

3

Docker

The static website is packaged into an Nginx Docker image and pushed to Docker Hub.

4

Argo CD

Argo CD detects the updated Kubernetes manifest and syncs the cluster automatically.
